# 颜色警报功能说明

## 功能概述

**颜色警报** 是 OnTopReplica 中新增的一个监控功能。当你在监控的窗口中检测到指定的颜色时，应用会自动发出3秒的报警声。

## 使用方法

### 1. 打开颜色警报面板

在主窗口的右键菜单中找到 **"Color Alert"** 选项，点击即可打开颜色警报面板。

### 2. 配置颜色检测

颜色警报面板包含以下设置：

#### **Enable Color Detection** （启用颜色检测）
- 复选框，用于启用/禁用颜色检测功能
- 默认为禁用状态

#### **Target Color** （目标颜色）
- 显示当前选定的目标颜色
- 点击 **"Choose..."** 按钮打开系统颜色对话框
- 支持从完整的颜色范围中选择任何颜色
- 右侧显示十六进制颜色值（如 #FF0000 代表红色）

#### **Color Tolerance** （颜色容差）
- 范围：0-255
- 数值越高，匹配范围越宽
- 例如：
  - 容差 = 0：只匹配完全相同的颜色
  - 容差 = 30：允许 RGB 各分量±30 的偏差（推荐值）
  - 容差 = 100：允许更大的颜色变化
- 使用滑块或直接输入调整

#### **Sample Interval** （采样间隔）
- 范围：100-10000 毫秒
- 两次颜色检测之间的时间间隔
- 值越小，检测越频繁但消耗 CPU 越多
- 值越大，检测更省资源但可能延迟检测
- 推荐值：300-1000 毫秒

### 3. 启动监控

1. 使用 OnTopReplica 克隆一个窗口到最前面
2. 在颜色警报面板中选择目标颜色
3. 初始化容差和采样间隔参数
4. 勾选 **"Enable Color Detection"** 复选框
5. 关闭面板返回主窗口

### 4. 警报触发

当检测到目标颜色时：
- 应用会自动播放3秒的报警声
- 报警声由系统蜂鸣音组成（约200ms间隔重复）
- 日志文件中会记录颜色检测事件

## 工作原理

1. **定期采样**：根据设置的采样间隔，定期获取被监控窗口的屏幕内容
2. **像素网格采样**：为了性能优化，应用不检查每个像素，而是采样一个 20x20 的网格
3. **颜色匹配**：对采样的像素进行颜色匹配，容差范围内的颜色都被认为是目标颜色
4. **警报触发**：检测到匹配后，播放3秒的报警声
5. **冷却期**：报警完成后，需要等到冷却期结束才能再次触发报警

## 使用场景

- **游戏监控**：监控特定颜色的敌人或资源，及时发出警告
- **数据监控**：监控应用界面中特定颜色的状态指示器变化
- **工业控制**：监控设备面板上的指示灯颜色变化
- **实时通知**：当网站或应用出现特定색颜色提示时发出警报

## 常见问题

**Q: 为什么有时候警报不会触发？**
A: 
- 检查 "Enable Color Detection" 是否勾选
- 检查颜色容差设置是否太小
- 确保目标颜色在被监控窗口中存在
- 检查采样间隔设置，如果过大可能延迟检测

**Q: 报警声太小/太大怎么办？**
A: 使用系统音量控制器调节应用音量。报警声使用系统 Beep 声音。

**Q: 会不会很耗 CPU？**
A: 颜色检测采用了优化措施：
- 采样网格化（而非逐像素检查）
- 根据采样间隔调节频率
- 建议采样间隔设置在 500ms 以上
- CPU 使用率通常在 1-5% 左右

**Q: 如何快速切换颜色？**
A: 
- 打开颜色警报面板
- 修改目标颜色
- 关闭面板，设置自动保存

## 性能建议

- 采样间隔设置为 500-1000ms 平衡检测速度和性能
- 颜色容差设置在 20-40 获得较好的识别效果
- 如果监控的窗口分辨率较高，可适当增加采样间隔

## 技术实现细节

- **消息处理器**：使用 Windows 消息泵设计
- **像素采样**：获取窗口客户区内容并进行网格采样
- **音频播放**：使用 System.Media.SystemSounds 播放报警音
- **线程安全**：所有操作都在应用主线程中进行
